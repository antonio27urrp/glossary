# Отчет по заданию: REST API «Глоссарий» на FastAPI

Данный репозиторий содержит реализацию сервиса **Глоссарий терминов** в соответствии с требованиями задания по изучению принципов REST и фреймворка FastAPI.

---

## Функциональные возможности

Сервис поддерживает полный цикл **CRUD-операций** для работы с терминами:

- **Получение списка всех терминов** — `GET /terms`
- **Поиск термина по ключевому слову** — `GET /terms/{keyword}`
- **Добавление нового термина с описанием** — `POST /terms`
- **Обновление существующего термина** — `PATCH /terms/{keyword}`
- **Удаление термина из глоссария** — `DELETE /terms/{keyword}`

---

## Технологический стек и особенности реализации

В ходе разработки были применены следующие инструменты и подходы:

- **FastAPI и Pydantic**  
  Использование Pydantic-схем обеспечивает строгую валидацию входных данных и автоматическое формирование JSON-ответов.

- **OpenAPI и Swagger**  
  Благодаря встроенной поддержке OpenAPI сервис автоматически генерирует интерактивную документацию, доступную по адресу `/docs`.

- **SQLite и SQLModel**  
  Для хранения данных используется легковесная база данных SQLite, работающая через библиотеку SQLModel.

- **Автоматическое создание схемы БД**  
  Структура таблиц создается автоматически при старте приложения с помощью команды `SQLModel.metadata.create_all`.

- **Frontend-визуализация**  
  Реализован интерактивный семантический граф терминов на React с использованием библиотеки `react-force-graph-2d`, визуализирующий связи между понятиями.

---

## Развертывание сервиса через Docker Compose

Решение реализовано в виде связки контейнеров, что обеспечивает переносимость и стабильность среды развертывания.

### Инструкция по запуску

1. Убедитесь, что в системе установлены **Docker** и **Docker Compose**.
2. В корневой папке проекта выполните команду:

```bash
docker-compose up --build
```

3. Дождитесь завершения сборки образов и появления сообщения о готовности серверов в терминале.

### Доступ к компонентам

- **Веб-интерфейс (граф терминов):** http://localhost:3000
- **Документация API (Swagger):** http://localhost:8000/docs
